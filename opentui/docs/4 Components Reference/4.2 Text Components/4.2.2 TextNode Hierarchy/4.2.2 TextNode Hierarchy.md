## 4.2.2 TextNode Hierarchy

The Solid reconciler distinguishes between regular renderables and text nodes for proper parent-child relationships: 

```mermaid
graph TB
    TextElement["&lt;text&gt; Element"]
    TextRenderable["TextRenderable Instance"]
    RootTextNode["RootTextNodeRenderable"]
    TextNodeChild1["TextNodeRenderable Child 1"]
    TextNodeChild2["TextNodeRenderable Child 2"]
    StringContent["String Content"]
    
    TextElement --> TextRenderable
    TextRenderable --> RootTextNode
    RootTextNode --> TextNodeChild1
    RootTextNode --> TextNodeChild2
    TextNodeChild1 --> StringContent
    
    style TextRenderable fill:#f9f9f9,stroke:#333,stroke-width:2px
    style RootTextNode fill:#f9f9f9,stroke:#333,stroke-width:2px
```

Text nodes created via `createTextNode` use a specialized `TextNode` class that extends `TextNodeRenderable`.  The reconciler enforces that text nodes must have a `TextRenderable` or another `TextNodeRenderable` as a parent, throwing an "Orphan text error" if this constraint is violated at .
