## SolidJS Integration

OpenTUI provides support for Solid.js through the `@opentui/solid` package.  This package acts as a Solid reconciler, allowing you to build terminal user interfaces using Solid.js components.  

### Installation 

To use Solid.js with OpenTUI, you need to install `solid-js` and `@opentui/solid`: 

```bash
bun install solid-js @opentui/solid
``` 

### Usage 

1.  **Configure JSX in `tsconfig.json`**: 
    Set `jsx` to `"preserve"` and `jsxImportSource` to `"@opentui/solid"`. 

    ```json
    {
      "compilerOptions": {
        "jsx": "preserve",
        "jsxImportSource": "@opentui/solid"
      }
    }
    ``` 

2.  **Add preload script to `bunfig.toml`**: 

    ```toml
    preload = ["@opentui/solid/preload"]
    ``` 

3.  **Render your Solid component**: 
    Use the `render` function from `@opentui/solid` to mount your Solid.js application. 

    ```tsx
    import { render } from "@opentui/solid"

    render(() => <text>Hello, World!</text>)
    ``` 

4.  **Run with Bun**: 
    Execute your application using `bun index.tsx`. 

5.  **Build with Bun.build**: 
    For building, you can use `Bun.build` with the `solidPlugin` from `@opentui/solid/bun-plugin`. 

    ```ts
    import solidPlugin from "@opentui/solid/bun-plugin"

    await Bun.build({
      entrypoints: ["./index.tsx"],
      target: "bun",
      outdir: "./build",
      plugins: [solidPlugin],
      compile: {
        target: "bun-darwin-arm64",
        outfile: "app-macos",
      },
    })
    ``` 

### Components 

OpenTUI Solid exposes intrinsic JSX elements that map to OpenTUI renderables.  These include:

*   **Layout & Display**: `text`, `box`, `scrollbox`, `ascii_font` 
*   **Input**: `input`, `textarea`, `select`, `tab_select` 
*   **Code & Diff**: `code`, `line_number`, `diff` 
*   **Text Modifiers**: `span`, `strong`, `b`, `em`, `i`, `u`, `br`, `a` (used inside `<text>` components) 

### API Reference 

The `@opentui/solid` package provides several functions and hooks for SolidJS integration:

*   `render(node, rendererOrConfig?)`: Renders a Solid component tree into a CLI renderer. 
*   `testRender(node, options?)`: Creates a test renderer for snapshots and interaction tests. 
*   `extend(components)`: Registers custom renderables as JSX intrinsic elements. 
*   `getComponentCatalogue()`: Returns the current component catalogue. 
*   **Hooks**: `useRenderer()`, `onResize(callback)`, `useTerminalDimensions()`, `useKeyboard(handler, options?)`, `usePaste(handler)`, `useSelectionHandler(handler)`, `useTimeline(options?)`. 
*   `Portal`: Renders children into a different mount node. 
*   `Dynamic`: Renders arbitrary intrinsic elements or components dynamically.
