## CliRenderer

The `CliRenderer` is described as the "heart of OpenTUI" and the central coordinator for the rendering system.   It manages terminal output, handles input events, and orchestrates the rendering loop. 

Key responsibilities of the `CliRenderer` include:
*   **Render Loop**: Orchestrates the rendering process, including `requestRender()` and `loop()` for frame throttling. 
*   **Tree Management**: Manages the `RootRenderable` and its children. 
*   **Input Handling**: Processes stdin listeners, `KeyHandler`, and `MouseParser`. 
*   **Layout Calculation**: Integrates with Yoga for layout tree traversal. 
*   **Event Dispatch**: Handles focus management and mouse hit testing. 

The `CliRenderer` class in TypeScript extends `EventEmitter` and implements `RenderContext`.  It holds a pointer to its native Zig counterpart (`rendererPtr`) and manages the `stdin` and `stdout` streams. 

A `CliRenderer` instance can be created using the `createCliRenderer` asynchronous function, which takes an optional `CliRendererConfig`.  This function initializes the Zig renderer, sets up Kitty keyboard protocol flags, and performs terminal setup. 

## RenderContext

`RenderContext` is an interface implemented by the `CliRenderer` class.  It is used to provide the renderer instance to `Renderable` components. For example, `ThreeRenderable` requires a `CliRenderer` context.  In framework integrations like SolidJS, the `RendererContext` is provided to components via a SolidJS `createContext` mechanism.
